WingWatch Development Process

Project Brief

WingWatch is an innovative website dedicated to revolutionizing the birdwatching experience in Brisbane. Our platform offers three key features designed to engage and educate bird enthusiasts:

  1. Advanced Bird Search: Utilize our sophisticated search engine to discover and identify birds native to Brisbane, leveraging comprehensive databases and user-friendly interfaces.
  2. Personal Digital Collections: Create and curate your own digital bird collection, a unique feature that sets us apart from other birdwatching websites. This allows users to track their sightings and build a personalized ornithological portfolio.
  3. Community Sharing and Interaction: Explore and appreciate collections shared by fellow birdwatching enthusiasts, fostering a vibrant community of bird lovers and promoting knowledge exchange.

function fetchOccurrences(map, centralLocation, radius) {
    showLoading();
    const dateRange = getDateRange(currentTimeframe);
    const pageSize = 100;
    const maxResults = 5000;
    let startIndex = 0;
    let allOccurrences = [];

    function fetchPage() {
        console.log(`Fetching page starting at index ${startIndex}...`);
        $.ajax({
            url: 'https://biocache-ws.ala.org.au/ws/occurrences/search',
            method: 'GET',
            data: {
                q: `*:*`,
                lat: centralLocation.lat,
                lon: centralLocation.lng,
                radius: radius.toFixed(2),
                qualityProfile: 'ALA',
                fq: [
                    'species_group:"Birds"',
                    `eventDate:[${dateRange.start}T00:00:00Z TO ${dateRange.end}T23:59:59Z]`
                ],
                pageSize: pageSize,
                startIndex: startIndex,
            },
            success: function (response) {
                console.log(`Fetched ${response.occurrences.length} occurrences from index ${startIndex}.`);
                allOccurrences = allOccurrences.concat(response.occurrences);
                if (startIndex + pageSize < Math.min(response.totalRecords, maxResults)) {
                    startIndex += pageSize;
                    fetchPage();
                } else {
                    console.log(`All pages fetched. Total records: ${allOccurrences.length}. Processing occurrences.`);
                    processAllOccurrences(allOccurrences, map, centralLocation);
                    hideLoading();
                }
            },
            error: function (xhr, status, error) {
                console.error("Error fetching data: ", status, error);
                hideLoading();
                alert("An error occurred while fetching data. Please try again.");
            }
        });
    }
    fetchPage();
}

Marker Filter Algorithm

Our user research revealed a critical insight: an overwhelming number of markers on the map can significantly degrade the user experience. In response, I've developed a sophisticated marker filter algorithm that limits the display to the five most relevant markers. This algorithm calculates the straight-line distance between the center of the map and each bird location, ensuring that only the closest and most pertinent data points are presented to the user.

This approach strikes a balance between providing comprehensive data and maintaining a clean, user-friendly interface. By focusing on the most relevant markers, I enhance the map's readability and reduce cognitive load for our users, allowing them to quickly identify and interact with the most significant bird sightings in their area of interest.

Distance Calculation Function

Algorithm to calculate distance between two geographical points

function calculateDistance(lat1, lon1, lat2, lon2) {
    const R = 6371; // Radius of the Earth in km
    const dLat = (lat2 - lat1) * Math.PI / 180;
    const dLon = (lon2 - lon1) * Math.PI / 180;
    const a = Math.sin(dLat / 2) * Math.sin(dLat / 2) +
        Math.cos(lat1 * Math.PI / 180) * Math.cos(lat2 * Math.PI / 180) *
        Math.sin(dLon / 2) * Math.sin(dLon / 2);
    const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
    return R * c; // Distance in km
}

Google Map API

The map loader is a self-executing function that sets up a mechanism to load the Google Maps JavaScript API dynamically. It creates a promise-based system to load the API script, handling various configuration options and library imports. The function sets up a global google.maps.importLibrary method, which allows for on-demand loading of specific Google Maps libraries. When called, it adds the requested library to a set, initiates the API loading process if not already started, and returns a promise that resolves when the library is ready. The code uses URL parameters to configure the API request, including the API key and version. It also handles error cases and ensures the API is only loaded once, even if the function is called multiple times with different libraries. This approach allows for efficient and flexible loading of Google Maps functionality in a web application.

Google Maps API Loader

Dynamic loading of Google Maps JavaScript API with custom configuration

(config => {
  // Declare variables for later use
  var apiLoadPromise, scriptElement, paramKey, 
      errorMessage = "The Google Maps JavaScript API",  // Descriptive string for error messages
      googleNamespace = "google",                      // Namespace for Google
      importLibraryFn = "importLibrary",               // Function name to import Google Maps libraries
      callbackName = "__ib__",                         // Internal callback property name
      doc = document,                                  // Reference to the document object
      win = window;                                    // Reference to the global window object

  // Initialize the Google namespace and maps object if not already defined
  win = win[googleNamespace] || (win[googleNamespace] = {});
  var mapsNamespace = win.maps || (win.maps = {}),
      librarySet = new Set(),                          // Set to store requested library names
      queryParams = new URLSearchParams,               // Create URLSearchParams object to build query string
      loadGoogleMapsAPI = () =>                        // Function to load the Google Maps JavaScript API
          apiLoadPromise || (apiLoadPromise = new Promise(async (resolve, reject) => {
              await (scriptElement = doc.createElement("script"));  // Create a new script element

              // Add library names to the URL parameters
              queryParams.set("libraries", [...librarySet] + "");
              
              // Add additional configuration options to the URL parameters
              for (paramKey in config) 
                  queryParams.set(paramKey.replace(/[A-Z]/g, match => "_" + match[0].toLowerCase()), config[paramKey]);

              // Set the callback parameter to the global callback function
              queryParams.set("callback", googleNamespace + ".maps." + callbackName);
              
              // Set the script source to the Google Maps API with the query parameters
              scriptElement.src = `https://maps.${googleNamespace}apis.com/maps/api/js?` + queryParams;

              // Define the global callback function to resolve the promise
              mapsNamespace[callbackName] = resolve;

              // Handle script loading error
              scriptElement.onerror = () => apiLoadPromise = reject(Error(errorMessage + " could not load."));

              // Set a nonce attribute if a script with a nonce exists in the document
              scriptElement.nonce = doc.querySelector("script[nonce]")?.nonce || "";

              // Append the script to the document head to start loading
              doc.head.append(scriptElement);
          }));

  // Check if the importLibrary function is already defined and warn if so; otherwise, define it
  mapsNamespace[importLibraryFn] ? console.warn(errorMessage + " only loads once. Ignoring:", config) :
         mapsNamespace[importLibraryFn] = (libraryName, ...rest) => librarySet.add(libraryName) && loadGoogleMapsAPI().then(() => mapsNamespace[importLibraryFn](libraryName, ...rest));
})

This code processes bird occurrence data and creates markers on a Google Map for each occurrence. The key part related to template literals is the creation of the infoWindowContent. A template literal is used to construct a complex HTML structure for the info window that appears when a marker is clicked. This multi-line string includes dynamic content (like ${commonName}, ${lat}, ${lon}, etc.) interpolated directly into the HTML structure. The template literal allows for easy embedding of variables and expressions within the string, making it simpler to create a rich, formatted content string with both static HTML and dynamic data. This approach is more readable and maintainable than traditional string concatenation, especially for complex HTML structures like this info window content.

Search Suggestion

This code implements an autocomplete feature for bird species search. Key points:

  • As users type, it filters common and scientific bird names, showing real-time suggestions.
  • Searches both common and scientific names, accommodating different user knowledge levels.
  • Users can click suggestions to auto-fill the search field.
  • Starts searching after two characters are typed for efficiency.
  • Provides visual feedback by showing/hiding suggestions dynamically.

Benefits for users:

  • Faster, more accurate species selection
  • Reduces spelling errors
  • Accommodates both casual and expert users
  • Speeds up search process
  • Educates users about bird names

Overall, it makes the bird search process more user-friendly, efficient, and informative.

Autocomplete Search Feature

Implementation of real-time search suggestions for bird species

$(document).ready(function() {
    const birdOccurrences = []; // This will hold all occurrences fetched from the API.
    scientificNames = window.scientificNames; // Populate scientific names
    commonNames = window.commonNames; // Populate common names

    // Function to update the suggestions list based on the user's input.
    function updateSuggestionsList(searchText) {
        // Check for matches in common names first
        let matchingNames = commonNames.filter(name => name.toLowerCase().startsWith(searchText.toLowerCase()));

        // If no matches found in common names, check scientific names
        if (matchingNames.length === 0) {
            matchingNames = scientificNames.filter(name => name.toLowerCase().startsWith(searchText.toLowerCase()));
        }

        console.log(`User input: ${searchText}`);
        console.log("Matching Names:", matchingNames);

        const suggestionsList = $('#suggestions-List');
        suggestionsList.empty();

        matchingNames.forEach(name => {
            suggestionsList.append(`<li>${name}</li>`);
        });
        
        // Hide suggestions if no matches found
        if (matchingNames.length === 0) {
            suggestionsList.hide();
        } else {
            suggestionsList.show();
        }
    }

    // Event handler for when the user types in the search input.
    $('#search-input').on('input', function() {
        const searchText = $(this).val();
        if (searchText.length >= 2) {
            updateSuggestionsList(searchText);
        } else {
            $('#suggestions-List').empty().hide();
        }
    });

    // Event handler for clicking on a suggestion.
    $('#suggestions-List').on('click', 'li', function() {
        const selectedName = $(this).text();
        $('#search-input').val(selectedName);
        $('#suggestions-List').empty().hide();
        $('#search-input').focus(); // Keep focus on the search input
    });

    // Event handler to detect click outside the input and list
    $(document).on('click', function (event) {
        const isClickInside = $(event.target).closest('#search-input, #suggestions-list').length > 0;

        if (!isClickInside) {
            const suggestionsList = $('#suggestions-list');

            // If there are items in the list, select the first one
            if (suggestionsList.children('li').length > 0) {
                const firstSuggestion = suggestionsList.children('li:first').text();
                $('#search-input').val(firstSuggestion);
            }
            suggestionsList.empty().hide(); // Hide the suggestions list
            $('#search-input').focus(); // Keep focus on the search input
        }
    });
});

Paper 1:Agile Work Practices and Team Performance

Paper: "Do agile work practices impact team performance through project commitment? Evidence from the information technology industry"

Authors: Uraon, R.S., Chauhan, A., Bharati, R. and Sahu, K.

Journal: International Journal of Productivity and Performance Management, Vol. 73 No. 4, pp. 1212-1234, 2024

DOI: 10.1108/IJPPM-03-2023-0114

Critical Evaluation:

This paper provides valuable insights into how agile work practices impact team performance in the IT industry. The authors conducted a comprehensive study involving 563 employees from 290 IT companies in India, examining the effects of agile taskwork and agile teamwork on team performance and project commitment.

Key Insights and Impact on Our Project:

  • Positive Impact on Team Performance: The study found that both agile taskwork and agile teamwork positively affect team performance. In our WingWatch project, implementing agile practices could potentially enhance our overall team performance by:
    • Improving our ability to respond quickly to changes in project requirements or user feedback
    • Enhancing our problem-solving capabilities through regular team discussions and retrospectives
    • Increasing our productivity through better task prioritization and time-boxed sprints
  • Project Commitment as a Mediator: The research shows that project commitment mediates the relationship between agile practices and team performance. This suggests that by adopting agile methodologies, we could increase team members' commitment to the project, which in turn would improve our performance. Specifically, this could lead to:
    • Higher levels of individual accountability through clearly defined sprint goals and daily stand-ups
    • Increased motivation as team members see regular progress and have more frequent opportunities to contribute
    • Stronger team cohesion as we collaborate more closely on short-term, achievable objectives
  • Agile Teamwork's Direct Impact: The paper indicates that agile teamwork has a direct positive effect on team performance, in addition to its indirect effect through project commitment. This underscores the importance of focusing on collaborative agile practices in our development process, such as:
    • Pair programming to improve code quality and knowledge sharing, particularly useful when integrating features like our search suggestions and time frame filter
    • Cross-functional teams to ensure diverse perspectives and skills are applied to each feature
    • Regular code reviews to catch integration issues early and maintain consistent coding standards
  • Career Level Considerations: Interestingly, the study found that career level negatively moderates the impact of agile practices on team performance. As our team consists of students at similar career levels, this might not significantly affect us, but it's worth considering for future projects with more diverse teams. We could:
    • Implement mentorship programs within the team to simulate different career levels and enhance knowledge transfer
    • Rotate roles and responsibilities to ensure everyone gains experience in different aspects of the project
    • Encourage continuous learning and skill development to mimic career progression within our academic setting

Application to WingWatch:

Given the challenges we faced with code integration and collaborative work, implementing agile practices could significantly benefit our project:

  • Improved Collaboration: Agile teamwork practices could help us better coordinate our efforts, reducing conflicts when multiple team members work on the same files.
  • Increased Project Commitment: By adopting agile methodologies, we could enhance team members' commitment to the project, potentially leading to more consistent and higher-quality contributions.
  • Enhanced Performance: The direct positive impact of agile practices on team performance suggests that we could see improvements in our overall productivity and output quality.
  • Iterative Development: Agile's focus on iterative development could help us identify and resolve integration issues earlier in the development process.

Moving forward, we should consider incorporating agile practices such as daily stand-ups, sprint planning, and regular retrospectives. These practices could help us better coordinate our efforts, increase project commitment, and ultimately improve our team performance and the quality of our WingWatch project.

Obstacles and Reflection

Throughout the development process, I encountered several challenges that provided valuable learning experiences. One of the most time-consuming features I worked on was changing the algorithm for filtering markers from straight-line distance to commute time.

Commute Time Algorithm Challenge

I attempted to use the Distance Matrix API (part of Google Maps API) to calculate commute times between points. However, I encountered a CORS (Cross-Origin Resource Sharing) error, which is essentially caused by server access control restrictions. After researching potential solutions, I tried several approaches:

  • Sending requests through a third-party proxy server, which worked initially but quickly reached access limits.
  • Building my own proxy server using Node.js, which introduced new bugs and was ultimately not allowed in the course.

Due to these obstacles, we had to revert to the original distance-based algorithm. A team member suggested using alternative mapping APIs like Leaflet, which could be explored in future iterations.

API Key Security

Similar difficulties arose when trying to securely hide API keys, as proper implementation typically requires a server-side solution. This reinforced the importance of considering security implications early in the development process.
